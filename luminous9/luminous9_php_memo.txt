■ LUMINOUS 9 - PHPフェーズ 引き継ぎメモ
========================================

【現在の状況】
HTML・CSS・JS フェーズ完成。
次はPHP・SQLによるランキング機能の実装。

----------------------------------------
【追加する機能】
----------------------------------------

1. ゲームクリア時にタイムをPHPへ送信
2. PHPがDBに保存
3. ランキング画面でDBから取り出して表示

----------------------------------------
【フォルダ構成（予定）】
----------------------------------------

LUMINOUS9/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── puzzles.js
│   └── script.js
├── images/
│   ├── logo.png
│   ├── timer.png
│   ├── flower_right.png
│   └── flower_left.png
├── php/
│   ├── save_score.php   ← タイムをDBに保存
│   └── get_ranking.php  ← ランキングをDBから取得
└── ranking.html         ← ランキング画面

----------------------------------------
【DB設計（予定）】
----------------------------------------

テーブル名：scores

| カラム名   | 型           | 内容               |
|-----------|-------------|-------------------|
| id        | INT AUTO_INCREMENT PRIMARY KEY | ID |
| difficulty | VARCHAR(10) | 難易度（easy/normal/hard/expert） |
| time_sec  | INT         | クリアタイム（秒）  |
| created_at | DATETIME   | 登録日時           |

----------------------------------------
【JS側の実装（script.js）】
----------------------------------------

checkClear()のクリア処理の中に以下を追加：

// クリアタイムをPHPへ送信
fetch('php/save_score.php', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    difficulty: gameState.difficulty,
    time_sec: timerSeconds
  })
});

----------------------------------------
【save_score.php の処理内容】
----------------------------------------

・POSTでdifficulty・time_secを受け取る
・DBに接続してscoresテーブルにINSERT
・成功/失敗をJSONで返す

----------------------------------------
【get_ranking.php の処理内容】
----------------------------------------

・難易度ごとにtime_secの昇順でSELECT
・上位10件をJSONで返す

----------------------------------------
【ランキング画面（ranking.html）】
----------------------------------------

・難易度タブで切り替え
・get_ranking.phpからfetchで取得して表示
・デザインはindex.htmlと統一

----------------------------------------
【注意事項】
----------------------------------------

・DBの接続情報は別ファイル（config.php）に分けて管理
・SQLインジェクション対策としてプリペアドステートメントを使用
